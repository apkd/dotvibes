Put together a short, descriptive commit message, followed by two newlines, then a longer commit description.

Check `git status --short` for a summary of changed files.
DO NOT re-read the diffs and files.

For more specific instructions, read `.../skills/commit/references/commit.md` if you haven't viewed that document before.
Skip otherwise.
